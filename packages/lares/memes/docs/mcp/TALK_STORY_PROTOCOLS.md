# Talk Story Protocols — MCP Crucible

Status date: **April 23, 2026**

This room tightens how the MCP shelf should draft, consume, and promote work.

---

## Core moves

### 1. Talk Story

Use talk story when:
- the shape still searches for its boundary
- several local sources need braid-reading before contraction
- metaphor, comparison, and provenance need to stay adjacent for one more pass

Talk story should produce:
- candidate distinctions
- local evidence trails
- named tensions
- bounded residue

Talk story should not pretend to finish contract work before the contract exists.

### 2. Refine-Consume

Refine-consume means:
- read broad source shelves
- extract the load-bearing braid
- compress it into matrices, contracts, and action tables
- mark what got consumed and what remains outward

Preferred outputs:
- inventories
- matrices
- boundary statements
- adapter contracts
- boot-preserved feature sets

### 3. Promote high-manaoio actions

High-manaoio actions carry these marks:
- local resolution can repeat reliably
- repo evidence already exists
- operator crossings stay bounded
- the action closes ambiguity rather than decorating it
- the output can feed implementation directly

Low-manaoio actions carry these marks:
- broad future-facing abstraction with weak repo contact
- platform-fashion pressure outruns local need
- hidden dependency on unstated manual interpretation
- promotion language outruns enacted work

## Promotion rule

When backlog pressure conflicts, prefer the ticket that:
1. consumes more local truth
2. sharpens more boundaries
3. lowers implementation ambiguity
