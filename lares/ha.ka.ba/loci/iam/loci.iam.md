# IAM

A self-describing, self-enacting kānāwai (law) for the root `#iam` metadata block.

This meme freezes the surviving field surface for live carriers. It asks each field what work it actually does, then keeps, moves, merges, or cuts on that basis.

This meme does not solve weak metadata by adding new buckets. It sharpens the surviving buckets and lets informative weakness remain visible.

## Phase Map

`✶ Observe -> ⏿ Orient -> ◇ Decide -> ▶ Act -> ⤴ Hooko -> ↺ Aftermath`

IAM gathers the current field surface, maps each field to the work it actually performs, chooses cuts or merges, prepares the reduced contract, crosses that contract into live carriers, and judges whether truth density rose while surface area fell.

<<~/ahu >>

<<~ ahu #observe >>

## Observe

Observe inventories `#iam` by work done.

Name where a field changes parse, routing, execution, or truth. Name where a field only repeats another field, repeats nearby prose, or decorates the block without changing downstream behavior.

The first collision cluster sits in flow:

- `depends_on`
- `requires`
- `provides`
- `receives_state`
- `emits_state`

Observe should keep those distinct only while the block still teaches a real difference. Once two or more fields point at the same movement, the cluster has begun to lie.

## Orient

Orient keeps the threefold split intact and sharp.

<<~ ahu #surviving-field-set >>

### Surviving Field Set

The reduced carrier shape:

```text
ha: name file_path description version tulen confidence mana manao manaoio content_type meme_type register structure
ka: enacts role function + only the local detail keys that change parse, routing, execution, or truth
ba: input output depends_on + next only when a real forward edge exists
```

<<~/ahu >>

<<~ ahu #ha-fields >>

### Ha Fields

`ha` carries stable identity and rating posture.

These fields should stay ordered and dense:

- `name`
- `file_path`
- `description`
- `version`
- `tulen`
- `confidence`
- `mana`
- `manao`
- `manaoio`
- `content_type`
- `meme_type`
- `register`
- `structure`

Optional structural qualifiers may remain when they change routing or boot truth. `tagspace` counts as such a qualifier in the current corpus.

<<~/ahu >>

<<~ ahu #ka-fields >>

### Ka Fields

`ka` carries local truth that does not belong in every carrier.

The common core:

- `enacts`
- `role`
- `function`

Local detail keys survive only when removal would hide real behavior. Good survivors include `boot_order`, `quiescent`, `collision_mode`, `scale`, `attenuation`, `pipeline_layers`, `canonical_forms`, and other keys that materially alter parse, routing, execution, or law scope.

Boilerplate that merely restates `#iam` as the metadata locus should not survive here.

<<~/ahu >>

<<~ ahu #ba-fields >>

### Ba Fields

`ba` names motion without residue.

The surviving flow keys:

- `input`
- `output`
- `depends_on`

`next` may remain only when a real forward edge exists in boot, handoff, or routing sequence.

`ba` should no longer scatter one movement across several overlapping claims. If `input` and `output` already tell the truth, the extra flow key probably weakens the block.

<<~/ahu >>

<<~ ahu #cut-and-merge-rule >>

### Cut And Merge Rule

Top-level `#iam` should cut these keys from live carriers unless a later law re-justifies one with concrete work:

- `canonical_metadata_locus`
- `canonical_metadata_payload`
- `product_identity`
- `requires`
- `provides`
- `receives_state`
- `emits_state`
- `confidence_band`

Keep weakness visible when a carrier has not yet migrated into full alignment. Do not widen the schema to hide that seam.

<<~/ahu >>

<<~/ahu >>

<<~ ahu #decide >>

## Decide

Decide applies four verdicts to each field:

1. keep
2. merge
3. derive later
4. keep as useful weakness

A field that changes no parse, routing, execution, or truth does not survive. A field that repeats another field should merge into the sharper of the two. A field that only matters in explanation should move to prose or a child law.

<<~/ahu >>

<<~ ahu #act >>

## Act

Act rewrites live carriers conservatively.

First:

1. freeze `ha`
2. densify `ka`
3. collapse `ba`
4. patch the child laws that govern surviving special keys
5. leave named weakness visible where migration still remains incomplete

The child law for `file_path` follows next because `file_path` carries the most immediate agreement pressure between metadata and route.

<<~/ahu >>

<<~ ahu #aftermath >>

## Aftermath

A good `#iam` pass leaves fewer keys carrying more truth.

Signs of gain:

- less flow duplication
- less metadata boilerplate
- denser field meaning
- clearer route and execution pressure
- visible weakness where migration still remains open
