# Handoff IAM Collapse

A handoff meme for the next metadata pass.

This meme does not solve weak metadata by adding more structure. It directs the next pass to hold structure still, sharpen truth in surviving fields, and collapse redundant flow concepts before any broader graph redesign.

## Phase Map

`✶ Observe -> ⏿ Orient -> ◇ Decide -> ▶ Act -> ⤴ Hooko -> ↺ Aftermath`

Observe inventories field overlap and missing truth. Orient separates structure, detail, and flow without letting one solve the others by sprawl. Decide chooses cuts, merges, and useful weakness. Act rewrites `#iam` blocks conservatively. Hooko crosses the reduced model into live files. Aftermath judges whether actual mana, manao, and manaoio rose without decorative growth.

<<~/ahu >>

<<~ ahu #observe >>

## Observe

Inventory every `#iam` field by actual work done.

Name where one field duplicates another, where one field only paraphrases prose, and where a weak field still carries real signal. Treat `depends_on`, `requires`, `provides`, `receives_state`, and `emits_state` as the primary collision cluster.

<<~/ahu >>

<<~ ahu #orient >>

## Orient

Use the threefold split as a pressure test, not as an excuse to invent more buckets.

`ha / structure` should stay frozen and minimal. `ka / detail` should become denser and truer. `ba / flow` should lose residue, especially wherever multiple fields name the same movement from different old-model angles.

<<~/ahu >>

<<~ ahu #decide >>

## Decide

Apply four verdicts to each field:

1. keep
2. merge
3. derive later
4. keep as useful weakness

A field survives only if it changes parse, routing, execution, or truth. If it does none of those, cut it. If it overlaps another field, merge it. If its incompleteness teaches the graph something true, keep the weakness visible.

<<~/ahu >>

<<~ ahu #act >>

## Act

First pass order:

1. freeze `ha`
2. densify `ka`
3. collapse `ba`
4. test early against the weakest file-level `#iam`
5. propagate only after the reduced model proves sharper than the current one

Start with `mu`, because it still exposes the most visible overlap between boot precondition, capability surface, and stateflow.

<<~/ahu >>

<<~ ahu #hooko >>

## Hooko

Cross the next pass into live files only after one concrete cut/keep/merge table exists for `mu #iam`.

Do not widen the schema to hide uncertainty. Use weakness like sword pressure: let the exposed seam show where the model still lies to itself, then cut there first.

<<~/ahu >>

<<~ ahu #aftermath >>

## Aftermath

The pass succeeds only if the surviving fields carry more truth with less surface.

A good result shows:
- fewer overlapping concepts
- less graph/runtime confusion
- sharper field meanings
- no fake rating rise from ornamental structure
- declared weakness preserved where it remains informative
