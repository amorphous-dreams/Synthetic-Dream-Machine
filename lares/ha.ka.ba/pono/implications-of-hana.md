## Canonical Decision

The current law stack should treat `hana` as:

* an admitted guest-work sigil
* block-only in canonical form
* optionally payload-block when the first meaningful child carries fenced `toml`
* distinct from the root primitive cluster
* the canonical worksite for guest grammars that perform data or code processing work

This decision implies that the current memes should stop leaning on inline guest-work examples and should instead present `hana` as a bounded worksite with separate priming payload and body matter.
