## Pressure

The recovered stack prefers vivid names for recurrent failures because vivid names travel further through a live session than abstract caution labels.

The recurring pattern:

- certainty collapse gets a name
- mode theater gets a name
- memory misuse gets a name
- recursion excess gets a name
- fiction-laundered failure gets a name

This naming practice reads as partly Satirist, partly operational hygiene.
It turns generic model failure into recognizable house weather.
Once named, it becomes easier to flag early and easier to discuss without false shame.
