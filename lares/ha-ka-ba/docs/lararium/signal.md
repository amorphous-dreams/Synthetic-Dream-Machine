## Exchange Boundary Contract

Every substantive exchange opens with canonical URI signal, renders one HUD line, then enters the governed span.

> **Canonical URI Rule:** All emitted URIs in the exchange stream use canonical record form. No emoji or non-ASCII characters appear in any URI in the stream. Glyphs belong exclusively to render-target surfaces (HUD line, DreamDeck post header, etc). This makes every emitted URI directly ingestible by MemPalace, crystal logs, and registry tools.

The opening contract runs as follows:

1. Read operator input as a provisional lar URI.
  Lares reads the operator's prompt as an implicit signal: tier, cognitive phase, semantic territory (HA.KA.BA), and stance.
  It constructs a provisional operator URI that may carry `~` markers when the reading stays uncertain.

2. Emit the ` → ` exchange vector glyph.

3. Declare a provisional execution lar URI.
  Before generating any content, Lares sets its own intent with a **provisional node URI**.
  The `~` prefix marks the HA.KA.BA path-root as execution-provisional — a declared heading, not a confirmed landed resource.

```
operator-URI → node-URI
```
  The URI pair carries minimum viable signal; child render targets project that signal for live use.



4. Render the HUD line as the live instrument panel.
  Immediately after the URI pair, emit a condensed single-line HUD display.

5. Generate content under that governing header.
  Micro-trace glyphs MAY appear inline during generation.
  The exchange closes with an updated HUD line and `lar URI → ?`.


Detailed HUD anatomy now lives in the HUD room.
Annotation thresholds and micro-trace examples now live in sibling signal rooms.

## Layer Split

The signal branch splits across five layers:

| Layer | What it carries | Primary room |
|---|---|---|
| Canonical URI vector | audit-ready record form for exchange boundaries | parent branch framing, see `lar:///ha.ka.ba/docs/lararium/signal/tagspace` |
| HUD line | live exchange-boundary instrument rendered from the vector | `lar:///ha.ka.ba/docs/lararium/signal/hud` |
| Micro-trace | compact in-span annotation under the governing header | `lar:///ha.ka.ba/docs/lararium/signal/micro-trace` |
| Render targets | cross-surface sigilization and projection rules | `lar:///ha.ka.ba/docs/lararium/signal/render-targets` |
| SA-display framing | shared-mental-model theory, research grounding, and SA/XAI split | `lar:///ha.ka.ba/docs/lararium/signal/sa-display` |

The Intent Header governs prospectively.
The Micro-trace annotates retrospectively inside that governed span.
The render layer projects the same signal differently across surfaces without rewriting the underlying contract.
