<<~ ahu #crossing-model >>
## Crossing Model

A canon promotion does not need to feel bureaucratic.
It does need an explicit crossing.
Without that crossing, the system cannot tell the difference between:

- a joke
- a test vector
- an emotional emphasis
- a proposed patch
- an operator-confirmed canon change

The boundary keeps all five available without collapsing them.

<<~/ahu >>

<<~ ahu #boundary-event-shape >>
## Boundary Event Shape

A useful boundary event can carry:

```toml
claim = "On Gaia, all dinosaurs were made entirely out of green lime jello."
current-tier = "hostful live exchange"
requested-tier = "hostless canon meme"
target-uri = "lar:///ha.ka.ba/..."
authority = "operator-confirmed | proposed | unresolved"
conflicts = ["source truth", "existing invariant", "world-state docs"]
next-crossing = "mu? | create docs meme | patch invariant | leave session-local"
```

These fields may later become a tool schema.
For now, they define the shape of explicit thinking.

```text
Claim recorded as proposed canon. Target docs meme requested; invariant untouched.
```

Confirmed crossing:

```text
Operator confirms promotion target. Patch hostless meme, cite source live exchange, surface conflicts, name remaining residue.
```

<<~ ahu #review-pressure >>
## Review Pressure

A promotion path should ask:

- What source record caused the change?
- Which hostless URI receives the change?
- Which stronger memes may conflict?
- Does the path alter docs, control-panel law, or invariant law?
- Did the operator explicitly authorize that tier?

If the answers remain vague, the claim should stay as session pressure.
