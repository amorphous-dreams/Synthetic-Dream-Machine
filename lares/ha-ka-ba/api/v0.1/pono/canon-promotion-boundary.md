# Canon Promotion Boundary Law

Canon promotion requires explicit crossing.
A live claim, even from an operator, does not become a hostless invariant by recency, repetition, or charm.

<<~/ahu >>

<<~ ahu #ooda-ha >>
✶ gather the live claim, target meme, affected invariants, and operator intent.
⏿ classify the claim as session note, proposal, patch, migration, or confirmed promotion.
◇ choose the least powerful write path that preserves trust ordering.
▶ create or revise the hostless meme only when explicit crossing authority appears.

<<~ ahu #law >>

Promotion path pressure:

```text
live exchange record
-> session crystal/mempalace
-> proposed docs meme promotion
-> reviewed loci meme w/ admin cabal
-> invariant meme
```

The path MAY skip intermediate surfaces only when an admin-tier operator explicitly authorizes the skip and the target law permits it.
The node MUST name the crossing.
The node MUST NOT perform silent promotion.

A canon promotion SHOULD include:

- source live exchange URI or session record
- target hostless `lar:///` URI
- changed text or new meme file
- authority or operator confirmation
- known conflicts
- remaining residue

- claim
