<<~ ahu #workflow-contract >>

## Workflow Contract

Use this skill to:

1. draft a new meme from the lawful copy-shape
2. audit an existing meme for conformance drift
3. compress an overloaded parent toward invariant form
4. split examples, research, or procedure into outward surfaces

Working order:

1. read the carrier fully
2. detect required shape first
3. classify optional depth separately
4. identify which outward type laws the carrier really satisfies
5. tighten the parent before creating sidecars
6. verify that the result still reads as one coherent meme

Do not:

- widen the parent with archive commentary
- hide missing required structure behind pretty prose
- treat a phase stub as filled content
- award canon posture because the file sounds confident
- spray outward links that carry no actual pressure

<<~ ahu #orient >>

## ⏿ Orient

Sort findings into these buckets:

- `required-shape`
- `optional-depth`
- `subtype-law`
- `procedure-or-skeleton`
- `research-or-docs-residue`

Orientation rule:

- `required-shape` stays nearest the parent
- `optional-depth` stays only if it sharpens the invariant
- `subtype-law` usually moves to the relevant outward law
- `procedure-or-skeleton` belongs in this skill
- `research-or-docs-residue` belongs in docs

<<~/ahu >>

<<~ ahu #decide >>

## ◇ Decide

Choose the smallest lawful shape:

- parent only, when copy-shape pressure already reads clearly
- parent + `SKILL.md`, when drafting or audit workflow needs room
- parent + docs, when examples, tables, or research crowd the root
- parent + `SKILL.md` + docs, when law, procedure, and explanation all need different rooms

Then choose one honest lifecycle posture:

- noise
- data
- meme
- typed meme
- canon typed meme

Do not round up.
