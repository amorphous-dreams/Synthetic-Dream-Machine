<<~ ahu #workflow-contract >>

## Workflow Contract

Use this skill to:

1. draft a guest-grammar law surface
2. audit host-guest boundary discipline
3. shape a lawful `hana` block
4. split parser, lowering, or fixture detail into outward surfaces

Working order:

1. read the carrier or target region fully
2. identify the host surface and the admitted guest site
3. classify material into admission law, hana shape, ownership boundary, degradation, lowering, or fixtures
4. keep only invariant admission law in the parent
5. move checklist, examples, and execution detail outward
6. verify that the host still reads clearly when the guest region fails

Do not:

- treat ambient punctuation as lawful guest admission
- let a root primitive quietly stand in for a guest family
- move canonical identity payload into `hana`
- hide malformed guest work behind a fake success path
- burden the parent with full parser or render implementation detail

## ⏿ Orient

Sort findings into these buckets:

- `admission-law`
- `hana-shape`
- `ownership-boundary`

- `parser-or-lowering-detail`
- `fixtures-or-docs-residue`

Orientation rule:

- `admission-law`, `hana-shape`, and `ownership-boundary` stay nearest the parent
- `degradation` stays only at the level of invariant fallback pressure

## ◇ Decide

Choose the smallest lawful shape:

- parent only, when admission law already reads clearly
- parent + `SKILL.md`, when audit or shaping procedure needs room
