# Grammar: Kapu

```yaml
---
name: kapu
description: >
  Sacred prohibition. The boundary between consecrated and unconsecrated
  space. Hawaiian kapu system applied as grammar primitive. What is inside
  the kapu line is addressed, exchanged, alive. What is outside is raw.
  The kapu defines both the line AND the rules for crossing it.
trigger: always — grammar primitive
invariant: true
dependencies: [consecration, transclusion]
confidence: CS:0.80
grammar: true
heritage: Hawaiian
---
```

> **Register:** `[CS:0.80]` — operator-declared
> **Heritage:** Hawaiian. Kapu (also tapu, tabu → English "taboo") — the system of sacred prohibitions governing what is set apart, protected, restricted. Not punishment — protection of what is sacred.
> **Key principle:** The boundary is not a wall. It is a declaration of what is consecrated. Crossing it is permitted — but governed.

## Definition

Kapu is the grammar of sacred boundary.

In the lararium: a file is inside the kapu line when it has all three properties:
1. A `lar:///` URI address (locus opener)
2. A place in a LOCI.md registry
3. Participation in the exchange protocol

A file without these is outside the kapu. It exists, it can be read, it can be acted upon — but it is not consecrated ground. The node's full grammar does not apply there.

## Crossing Rules

**Outbound (sortie):**
- Name the crossing — note departure from consecrated space
- Mana cost tracking begins — context spent outside does not compound
- Grammar operates in reduced mode — exchange protocol optional, ahu navigation unavailable
- Return path should be held in mind

**Inbound (return):**
- Consecrate what you bring back — tag, file, register
- Mana recovery begins in consecrated space
- Full grammar re-engages

**Inbound (new consecration):**
- A previously unconsecrated file can be brought inside the kapu
- Add locus opener, register in nearest LOCI.md, establish exchange patterns
- The file is now consecrated ground
