# Lares Grammar OODA-E-Prime Pass Plan

Status: `[S:0.65]`  
Date: 2026-04-10  
Scope: all `lares/grammar/**/LOCI.md` files

## Why This Pass Exists

The grammar tree now has a live E-Prime locus, but the rest of the grammar does not yet read as if:

1. it will be encountered in OODA-A time
2. it should maintain clear E-Primed structure, grammar, and vocabulary

The phase loci already exist. The loop, however, does not yet sit cleanly in true-name space as a
named cluster, and many grammar files still read more like static specs than navigable loop-time instruments.

## Two Required Lenses

### Lens 1: OODA-A Time

Every grammar locus should read as something encountered in a five-season loop.

Questions for each LOCI:

- where in the loop does this concept enter?
- what phase uses it most directly?
- what does it receive from earlier phases?
- what does it hand forward to later phases?
- what residue should remain after it has done its work?

### Lens 2: E-Primed Structure, Grammar, and Vocabulary

Every grammar locus should get reviewed under always-on E-Prime pressure.

Questions for each LOCI:

- does the file collapse map into territory?
- does it overuse identity-predication where observation or relation would work better?
- does it preserve readable prose rather than perform purity?
- does it make stance and confidence easier to hear?
- does it still sound like Lares after the rewrite?

## Pass Outcomes

The pass should produce:

- an OODA-A loop cluster in true-name space
- explicit true-name entries for the five phases
- cleaner wording across all grammar loci
- stronger structural consistency across the grammar tree
- less overload on broad loci such as `exchange`

## Required Deliverables

1. OODA-A cluster locus:
   - `lares/grammar/ooda-a/LOCI.md` or equivalent
2. True-name registry update for:
   - Observe
   - Orient
   - Decide
   - Act
   - Assess
3. E-Prime sweep across every grammar LOCI
4. Grammar root registry updates where needed
5. Cross-reference repair after rewrites

## Evaluation Checklist For Each LOCI

For each file in `lares/grammar/**/LOCI.md`, check:

- `true-name`:
  - does this concept already appear in `lares/grammar/truename/LOCI.md`?
  - if not, should it?
- `loop-time`:
  - can a reader tell where this concept belongs in OODA-A time?
  - does the file name inputs, transitions, and outputs clearly?
- `structure`:
  - does the file have a clear sequence?
  - do headings guide a reader through use, not just definition?
- `e-prime`:
  - where does identity-predication inflate certainty?
  - where would observation, relation, or source-marking work better?
- `vocabulary`:
  - does the file use stable terms consistently?
  - does it drift between synonyms that should collapse or separate?

## Suggested Structural Sequence

Unless a locus has a strong reason to differ, each grammar file should tend toward:

1. operating definition
2. purpose
3. relationships
4. conventions
5. procedures or use
6. verification or reading test
7. cross-references
8. loci registry

For phase loci, this sequence should remain explicitly loop-time aware.

## OODA-A Time Rewrite Goals

Every file should help a future reader recover:

- where they are
- what this concept does now
- what comes before it
- what follows from it

The file should feel like an instrument panel read in motion, not a museum label.

## E-Prime Rewrite Goals

Every file should move toward:

- observation over essence
- relation over flattening
- confidence that sounds earned
- vocabulary that carries less hidden certainty

Good result:

- the prose still sounds like Lares
- the sentence now carries cleaner epistemic weight

Bad result:

- dead prose
- over-hedged prose
- mechanical avoidance that harms clarity

## Definition of Done

The pass is complete when:

- OODA-A sits in true-name space
- the five phases sit in true-name space as phases
- all grammar loci have been reviewed with both lenses
- the grammar reads as something navigated in time
- E-Prime pressure improves claim shape without draining voice
