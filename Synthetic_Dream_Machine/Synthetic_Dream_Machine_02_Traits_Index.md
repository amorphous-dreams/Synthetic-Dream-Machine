# SDM Trait Template

*(Canonical, tag-forward, Markdown-friendly. This section defines the **schema only**. It contains no rules text, examples, or setting-specific guidance.)*

```text
#### **<Trait Name>**

text:
>   <Full canonical trait text, including the bolded title and all rules text, but excluding any numeric list index.>

tags:
>   [trait]
>   [type:<background|path|corruption|power|mutation|prosthetic|skillpack|other>]
>   [path:<Wizard|Vampire Lord|Traveler|…>]   # if applicable; spaces are fine
>   [corruption:<mild|moderate|severe>]       # if applicable
>   [affects:<attack|defense|save|reaction|inventory|actions|powers|social|…>]
>   [uses:<heroDie|life|abilityPoint|item|…>] # resource hooks, if any
>   [frequency:<once|per-round|per-scene|…>]  # if relevant
>   [inventory:<slot|free|modifies|…>]        # if it alters slot rules
>   [ref:<stable-id>]

meta:
> source:
> - book: <BookName>; page: <##>
> - book: <OtherBook>; page: <##>
>
> usage:
> - scope: <path|trope|corruption|powerlist|other>; id: <Wizard|Humans of the Pananthropy|…>; index: <##>; note: <extra context>
> - scope: <path|trope|corruption|powerlist|other>; id: <Alternate list>; note: <optional note about reuse>
```
Trait Notes:
- `text:` should capture the most recent/full version; `usage` explains how it extends earlier sources.
- Duplicate `source` entries whenever multiple books or paths continue to reference this trait.

Use the `meta` list to capture every structured placement of the trait (path slot, trope variant, corruption tier, etc.). Keep the `tags:` short and searchable—one `[path:slug]` (or analogous) tag per relevant list—and push 'index' order, die results, or other metadata into `meta`. Slugs can mirror the published names (whitespace included) as long as they stay consistent.

## Traits in Play

Traits represent the "radically ungated advancement" of the Sythetic Dream Machine. While clustered into "Paths" (tropes and themes), the players and referee are encouraged to mix-and-match freely, and to invent their own trait. Once a trait is written down on a character/item/vehile, it can also change during play! This index represents the "most common known variants".
